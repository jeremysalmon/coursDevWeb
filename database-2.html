Base de données

UML, Merise, OMT, ... différentes méthodes ... différents contextes

La modélisation de la base de données étant le socle des applications, de nombreuses méthodologies existent permettant à des équipes plus ou moins nombreuses de parler le même language quand il s'agit de la base de données.

Ce cours n'a pas pour objectif de vous apprendre une méthode particulière. Notre objectif ne sera donc pas d'être rigoureux quand à une méthodologie particulière mais plutôt de réussir à vous montrer la logique vous permettant de mettre en place des structures de bases solides.

Les étapes que nous devons suivre sont

  1. Dégager les entités de l'applications qui deviendront nos tables : (ex : facture, client, note, professeur, classe ...)
  2. Pour chaque tables, lister les champs : (ex : pour une table classe on imaginera : nom, description, date début, date fin, ...).
  3. Pour chaque champs définir son type. Ils existe de nombreux types de base de champs mais pour notre cours nous ne retiendrons que
    • INT : nombre entier positif ou négatif (ex : -10,42,-12365,465852). La valeur minimale est -2147483648, la valeur maximale est 2147483647
    • FLOAT : nombre à virgule (ex : 3.14, 10.23, 196540,45655856)
    • TEXT : Texte (64.000 caractères max)
    • DATETIME : Date et Heure au format AAAA-MM-JJ HH:MM:SS
  4. Pour chaque table, définir les relations (cardinalités) qu'elle entretient avec les autres tables. Ici pour notre cours nous retiendrons deux types de relations :
    • 1 à 1 : une facture appartient à un et un seul client, un client possède de 0 à N facture
    • 0 à N : une facture comporte de 0 à N produit, et un produit fait partie de 0 à N facture
