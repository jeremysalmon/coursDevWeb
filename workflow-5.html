Définir son projet - Les "User Stories"

Présentez vos fiches

Les users stories ne doivent pas être en langage technique. Vous devez faire abstraction des aspects techniques et vous concentrez sur les besoins utilisateurs. Vous pourrez ensuite écarter des User Stories s'il s'avère qu'elles sont techniquement difficile à réaliser.

  • Une catégorie : administration, utilisateur, backoffice facturation, etc ....
  • Un module : compte, produit, message
  • Une date : la date de création de cette User Storie
  • Un Titre : Création de compte, suppression de compte, ajout d'un nouveau produit ...
  • Une Description : ce que contient cette user storie par exemple
    "Quand l'utilisateur clique sur le bouton ajouter produit, une popup s'affiche. Quand il saisie le nom de son produit, il y a un système d'auto-completion (comme sur Google) qui lui indique les produits ressemblant existants. Quand le produit est créé, le système envoie un mail au service logistique qui est prévenu de la création du produit et qui peut alors commencer a saisir le stock dans le logiciel"

Vous pouvez par exemple choisir des feuilles Bristol de couleur différentes pour chaque catégorie.

Vous pouvez ensuite disposez au mur les User Stories par catégorie pour facilement avoir une visualisation
